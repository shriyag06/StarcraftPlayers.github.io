Starcraft is an esports title, and the business scenario involves assessing player rankings within the game by considering a range of factors.

Data Validation/ Data Cleaing Techniques used


1) Data Inconsistency

In the dataset, certain columns contained inconsistent data, including symbols like '?'. To address this issue, we initially replaced these symbols with 'NaN' values and subsequently filled these missing values with the median, especially since these variables exhibited a right-skewed distribution.


2) Data Transformation

To rectify the right-skewed nature of the data, appropriate transformations like taking the cube root, square root, applying logarithm, and utilizing the Box-Cox method were implemented.The illustration displays the degree of skewness within one of the characteristics.Skewed data is problematic in data analytics as it can lead to biased predictions, violate statistical assumptions, complicate model performance, distort descriptive statistics, and hinder decision-making, ultimately undermining the reliability and accuracy of insights derived from the data. Addressing skewness through suitable transformations or using robust modeling approaches is crucial to mitigate these issues.


3) Feature Selection

Two techniques were employed for feature selection: L1 Regularization and Information Gain. These approaches assisted in pinpointing the top 10 essential features for forecasting player ranks. After feature selection, standardization was applied to ensure uniform scaling across variables, preventing potential biases stemming from differing measurement units.


4) Handling Imbalanced Class

To address the imbalance in the target class, which corresponds to various player ranks, oversampling methods were utilized to create extra instances of the minority classes. This strategy proved to be more efficient than both SMOTE and undersampling approaches.The League index, serving as the target variable, consists of eight unique classes, with the count of instances in each class visible.
